# Releasing

Releases are cut locally (there is no CI publishing). The library crates are
published to crates.io together in lockstep at a single shared version (the two
`glassdb-bench-*` crates stay private). Each release also gets a single `vX.Y.Z`
git tag and a GitHub release whose notes are auto-generated by GitHub, so no
`CHANGELOG.md` is committed. The lockstep version bump is driven by
[release-plz](https://release-plz.dev) (configured in `release-plz.toml`); the
publish, tag, and GitHub release are done with `cargo publish --workspace`,
`git tag`, and `gh`.

## Prerequisites

- `CARGO_REGISTRY_TOKEN` set to a crates.io API token with publish rights.
- The GitHub CLI (`gh`) installed and authenticated (`gh auth login`).
- `release-plz` (auto-installed via `cargo install` on first run).

## Cutting a release

`hack/release.sh` is a dry run by default (it changes nothing); publishing
requires `--no-dry-run`.

```bash
hack/release.sh               # preview the bump + publish, changing nothing
hack/release.sh --no-dry-run  # bump, publish to crates.io, tag, and open the GitHub release
```

The script:

1. checks the working tree is clean and on `main`, with the tooling and
   credentials above present;
2. runs the `make test-all` gate;
3. bumps every published crate in lockstep with `release-plz update`;
4. shows the diff and asks for confirmation;
5. commits and pushes the version bump;
6. publishes to crates.io with `cargo publish --workspace`;
7. creates and pushes the single `vX.Y.Z` git tag;
8. opens the GitHub release with `gh release create --generate-notes`.

The version bump is computed from the conventional-commit history. To force an
exact version, call the script directly:

```bash
hack/release.sh --no-dry-run --version 0.2.0
```

Other flags: `--no-dry-run` (actually publish; otherwise it only previews) and
`--yes` (skip the confirmation prompt). See `hack/release.sh --help`.
